## 维护约定（提交代码时）

1. **根目录 `README.md`**：出现新的一级子项目目录、或某子项目定位/名称有变时，更新上表中的「一句话说明」与「详细文档」链接，保证与仓库实际结构一致。
2. **子目录 `README.md`**：每个承载业务的一级子目录均应有自己的 `README.md`（用途、依赖、使用步骤或当前规划）；新建目录时随代码一并补齐。
3. 向 GitHub **提交**前，对照本索引检查上述两项是否已更新，避免「有目录无说明」。
